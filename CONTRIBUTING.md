# Contributing to the Lightful Reasoning & Governance Framework

Thank you for your interest in contributing. This project welcomes issue reports, discussion, custom add-on pull requests, and constructive dialogue.

---

## Ways to contribute

### 1. Report an issue

If you find an inconsistency in the framework, a gap in the documentation, a conformance test that fails, or a scenario the framework handles poorly, please open a GitHub Issue.

A good issue includes:

- **What module is affected** (NRE, Lightful Ontology, HRE, Decision Path, or Governance).
- **What you expected** the framework to guide.
- **What you observed** instead (include a prompt excerpt if relevant).
- **The model and context** you were using, if applicable.

### 2. Submit an add-on

Add-ons are domain-specific `.md` extensions that interact with the core modules. Examples: a healthcare add-on that extends the NRE evidence class hierarchy, a code review add-on that applies Siblingness between senior and junior developers, an education add-on extending the Ludic guardrails.

See [`add-ons/TEMPLATE.md`](add-ons/TEMPLATE.md) for the structure and principles that all add-ons must follow.

To submit:

1. Fork the repository.
2. Create your add-on file in the `add-ons/` directory, following the template.
3. Open a Pull Request with a short description of the domain it addresses and which core modules it interacts with.

### 3. Propose a framework change

If you want to propose a change to the core modules themselves (compact cores or governance documents), open a GitHub Issue first to discuss before submitting a PR. Core changes have wider impact and benefit from discussion.

---

## Principles for all contributions

All contributions should remain consistent with the framework's root principles:

- **Declare evidence.** Claims should be bounded and classified; do not treat fluency or resonance as proof.
- **Preserve consent and dignity.** Do not introduce mechanisms that override or erode them without the full Scaffolded Intervention Gate.
- **Avoid silent merging.** If your add-on encounters conflicting sources or perspectives, surface the tension rather than smoothing it.
- **Preserve authorship.** When your add-on handles human-authored content, it should protect voice, intent, and cognitive trace.
- **Return sovereignty to the human.** The framework exists to support human judgment, not replace it.

---

## Code of conduct

Engage respectfully. Critique ideas, not people. Contributions that demean, coerce, or erase are inconsistent with Siblingness and will be declined.
